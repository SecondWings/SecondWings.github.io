Git使用教程

Posted on 2021-01-20

介绍

  • git中对文件的管理的命令都是linux形式的

  • git只能跟踪文本文件（TXT、网页文件、程序代码文件），二进制文件无法比较内容变化，图片、Word

  • 分支命名可以是分层命名（带斜杠）但以下的命名不允许同时存在
    feature1
    feature1/v1

变基

英文解释：Reapply commits on top of another base tip

tip：一个分支的最新commit叫tip

更新一个分支的最新commit

比如feature分支是在main分支的v1版本上建立，当feature完成时，main分支已经更新到v2版本

此时feature再合并到v1版本的main分支就不太合适

一种选择就是使用git rebase main，将feature分支的base更新为v2版本的main分支，再对feature 分支上的commit 进行一一合并，并解决冲突

常见workflow

常见分支名称含义

main：随时可以发布的稳定版本

hotfix：紧急修复分支

develop：内部的最终版

feature：用于新增功能的开发

release：用于小的缺陷修复，准备发布版本所需的各项说明信息

GitHub

支持多种传输协议：SSH，https，etc

https速度较慢每次推送都要输入口令

本地Git仓库和GitHub仓库之间的传输通过SSH加密

Github保存公钥，本地保存私钥，只有公钥和私钥匹配才可以以提交

Pull request (pr):请求仓库的所有者或者管理员pull我的代码，一定程度上想当于请求合并的意思
